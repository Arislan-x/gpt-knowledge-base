# GPT 知识库 v1.3.8

`v1.3.8` 是一次本地持久化升级：新增“归档到本地”，让浏览器备份和导入内容可以写入用户主动授权的本地文件夹，并在缓存丢失后通过旧归档文件夹恢复。

## 新增功能

- 新增“归档到本地”：在工作站设置中开启，选择并授权本地文件夹后，会话会写入 `conversations/<平台>/...json` 和根目录 `index.json`。
- 工作站和插件弹窗打开时都会触发归档同步；打开期间如果浏览器本地备份变化，也会继续同步。
- 左上角显示 `{percent}%已归档到本地`，设置面板显示开启状态、权限状态和位置。
- 重新选择以前的归档文件夹时，工作站会自动读取旧归档，便于扩展被移除、浏览器缓存丢失或重装后恢复。

## 导入改进

- 文件、ZIP、文件夹和 ChatGPT 官方导出导入后，默认写入已授权的本地归档文件夹。
- 同一会话在浏览器备份、本地归档和导入文件中重复出现时，保留更新时间最新的版本。
- 未开启或未授权“归档到本地”时，导入仍按旧逻辑只在当前工作站临时读取。

## 界面修复

- 长会话右侧问题进度条改为在可用空间内滚动，并隐藏滚动条，避免越界。
- 平台筛选行最左侧新增“全部”入口。
- 修复英文工作站中本地归档来源标签未完全翻译的问题；现在显示为 `Folder: Archive locally`。

## 数据与权限

- 没有新增 Manifest 权限。
- 本地归档文件夹必须由用户通过浏览器文件夹选择器主动授权；扩展不能静默读取任意本地路径。
- 对话数据仍不上传开发者服务器。

## 更新方式

1. Chrome Web Store 用户会由 Chrome 自动更新到 `1.3.8`。
2. GitHub 离线安装用户下载 `gpt-knowledge-base-1.3.8.zip`；`.sha256` 文件仅用于可选完整性校验。
3. 将 ZIP 完整解压到固定文件夹，并在 Chrome 扩展管理页重新加载扩展。
4. 刷新已打开的 AI 会话页面，使新版内容脚本生效。

Chrome Web Store 更新包与 GitHub Release 中的 `gpt-knowledge-base-1.3.8.zip` 使用相同代码。
